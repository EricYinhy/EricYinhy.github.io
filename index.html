Pwnagotchi折腾记

2025-08-14

刚放暑假的时候，大概7月份，我看到了一个很好玩的项目，Pwnagotchi
这是一个电子宠物，类似于拓麻歌子，但是它实际上是一个用于wifi破解的的网络安全工具。
当时就觉得很有意思，想做一个玩玩。奈何囊中羞涩，想省点钱，之后再买。
到了前两天，有马内了，我便开始了我的折腾之路。

#材料与配件
一个waveshare的2.13寸墨水屏(v4)
raspberry pi zero wh(wh是带排针的版本，和zero w其实没啥区别)
一个ups模块
还有一张tf卡
以及，勇于折腾的精神（doge）

#第一回折腾:OTG(On_To_Go)驱动

收到快递之后我就兴冲冲的刷写固件，放到rasp里运行！

紧接着ssh连接树莓派

诶怎么没反应
哦原来是忘记打驱动了，电脑没有成功把树莓派识别为网络设备

上网下载安装好驱动，搞定！了吗？

屏幕并没有点亮

#第二回折腾:屏幕
能正常运行，能看到指示灯闪烁，就是不知道画面是啥样！
这就很难受了，屏幕用不上
我仔细看了看固件，是2021年更新的
我又看了看支持的屏幕，发现并没说支持v4!
这就很麻烦了

不过，这样的问题肯定不是我一个人遇到，所以，我准备向前人借鉴：查看issue!

我稍加搜索，就发现确实不止有一个人提交了关于v4屏幕不适配问题的issue，仔细观察之下发现有人提交了更新，是对v4屏幕做了支持。

我就纳闷了，有对v4屏幕的支持，为啥屏幕点不亮？
一看，原来是2022年的更新，但是最新版本的固件在2021年，完全没跟上！

这期间我看到了有人说第三方固件好用，有人说用xx.xx.xx的成功了。但我就是不知道怎么下手这个问题

于是我去问了ai,deepseek。我开了联网搜索模式。deepseek给我讲了一大堆解决方法，但是好多资料来自于gitee而不是github.
正当我在想这些有没有用时,我翻看了deepseek搜索到的几个网页，其中有一个发送在b站的图文教程。
我点了进去，这好像是我之前看的那个教程，但是因为资料不全最后没成功。这难道就是当时缺失的资料吗？
这篇文章解说如何对v3屏幕进行支持。我看到这个up的视频里有说v3的驱动兼容v4.
我感觉我抓到了救命稻草！这不就是解决方案吗?!太好了！
原文_来自b站up主“太阳拎着月亮”
文章主要的内容就是把master branch的代码下载，然后进行修改，在导入到pwnagotchi中，然后删掉pwnagotchi里原来的，再替换上新的。
我按照它说的做了，但是屏幕还是没有点亮。
此时我已经耗费了很多时间，只能放到明天再考虑了。

#第三回折腾:偷梁换柱
第二天，我再次翻阅github上的issue,再看昨天的那个文档。
我发现一件事，这个up主下载的源代码版本比我下载的要低。
也就是说，我更改代码再放入的行为是完全没必要的！因为在后续已经更新了！可以直接放入！
而且up主的文章也提供了很好的思路，那就是“偷梁换柱”，我在不修改代码的情况下直接放入，应该就能成功。
之后不出所料，成功点亮！

折腾好像结束了….吗？我还在继续折腾，我到现在只是成功的配置了pwnagotchi的基础设置。后面能探索的东西还有很多！现在pwnagotchi正在我电脑旁运行着，等待着我的进一步开发和探索。

折腾永无止境，人类就是在折腾中不断学习。好了！现在，不折腾了，睡觉！

咳咳，切回正题，那SQLMAP是干啥的捏？

首先要知道SQL是啥：
SQL（结构化查询语言）

那SQL是为谁服务的呢？
比如说，你在一个网站注册了一个用户，那网站把用户的名称和密码放在一个数据库里
诶，对，SQL就是专为数据库开发的语言，方便操作数据库

SQLMAP就是干一些坏事，往数据库里注入有害的代码或者读取用户表单等等
SQL注入的本质是把用户输入的数据当作代码来执行，那么怎样才能成功的注入呢？

首先要找到注入点：
在GET参数、POST参数、Cookie、Referer等地方尝试插入代码、符号或语句，尝试是否存在数据库参数读取行为

有这样几个注入点（参考知乎）：
get注入
在get传参时写入参数，将SQl语句闭合，后面加写入自己的SQL语句。
post注入
通过post传参，原理与get一样，重要的是判断我们所输入的信息是否与数据库产生交互，其次判断SQL语句是如何闭合的。
有些网站通过查询cookie判断用户是否登录，需要与数据库进行交互，我们可以修改cookie的值，查找我们所需要的东西。或者通过报错注入是网页返回报错信息。
Referer注入
Referer正确写法应该是Referrer,因为http规定时写错只能将错就错，有些网站会记录ip和访问路径，例如百度就是通过Referer来统计网站流量，我们将访问路径进行SQL注入，同样也可以得到想要的信息。
XFF注入
在用户登录注册模块在 HTTP 头信息添加 X-Forwarded-for: 9.9.9.9’ ，用户在注册的时候，如果存在安全隐 患，会出现错误页面或者报错。从而导致注册或者登录用户失败。

然后用扫描工具对数据库进行扫描：
